INTRODUCTION To C PROGRAMMING PARADIGMS

Procedural Programming: This paradigm focuses on breaking down a program into smaller, reusable functions, which are executed sequentially. It emphasizes step-by-step instructions to achieve a specific task.
Structured Programming: A subset of procedural programming, structured programming emphasizes the use of structured control flow constructs like loops and conditional statements to enhance the clarity and maintainability of code.
Object-Oriented Programming (OOP): OOP is a paradigm that uses objects to represent real-world entities and encapsulates data and behavior within these objects. It promotes concepts like inheritance, polymorphism, and encapsulation to organize and manage code efficiently.
Functional Programming: In this paradigm, programs are built using functions as the primary building blocks. Functions are treated as first-class citizens, allowing for higher-order functions and functional composition.

data types in c

Basic Data Types: char: Represents a single character (usually 1 byte). int: Represents integers (whole numbers) and typically uses 4 bytes. float: Represents single-precision floating-point numbers (real numbers with decimal points) using 4 bytes. double: Represents double-precision floating-point numbers, providing more significant digits than float, and uses 8 bytes.
Derived Data Types: Arrays: A collection of elements of the same data type stored in contiguous memory locations. Pointers: Variables that store memory addresses, allowing manipulation of memory directly. Structures: User-defined data types that group together variables of different data types under one name. Unions: Similar to structures, but all members share the same memory location to store different data types.
Enumeration Data Type: enum: Allows the creation of a new data type, defining a set of named constant integer values.
Void Data Type: Represents the absence of a specific type. Used as a return type for functions that do not return a value or to declare generic pointers.

Constants in c programming

Constants in C are fixed values that remain unchanged throughout the execution of your program. They are used to represent specific values that you want to use in your code, like numbers or characters, but you don't want these values to change while your program is running. Constants are helpful because they make your code more readable and maintainable.
  • Integer constants: 10, 0, -5
  • Floating-point constants: 3.14, -2.5
  • Character constants: 'A', 'x', '1'
  • String constants: "Hello",
    "C programming"
  • Symbolic Constants: Also known as named constants, these are user-defined constants represented by identifiers (names) rather than fixed values. Symbolic constants are typically defined using the #define preprocessor directive or the const keyword.

enumeration constant in c

In C, enumeration constants (also known as enum constants) are a user-defined data type that allows you to create a set of named constant integer values. Enumerations provide a way to associate meaningful names to a set of related integer constants, making the code more readable and self-explanatory.

The syntax for defining an enum in C is as follows:

enum enum_name {
    constant1,
    constant2,
    constant3,
    // more constants...
};

Operator Precedence

Operator precedence in C refers to the order in which different operators are evaluated when an expression contains multiple operators. Operators with higher precedence are evaluated before operators with lower precedence. This is similar to how multiplication is done before addition in mathematics. 
For example, in the expression 3 + 5 * 2,
the multiplication (*) has higher 
precedence than addition (+), so
the multiplication is
done first: 5 * 2 = 10, 
and then the addition:
3 + 10 = 13.

Associativity in C

Associativity comes into play when an expression has multiple operators with the same precedence. It determines the order in which these operators are evaluated. C operators can be left-associative or right-associative.
*Most operators in C are left-associative, meaning they are evaluated from left to right.
For example, in the expression a + b + c, 
the addition is evaluated first for a + b,
 then the result is added to c.
*There are a few right-associative operators in C 
 such as the assignment operators (=), 
exponentiation (^), and the 
conditional operator (?:).

looping statements in c

The three main looping statements in C: for, while, and do-while, along with examples for each.
1.for Loop: The for loop is used for executing a block of code repeatedly for a specific number of times. It consists of an initialization, a condition, and an increment or decrement. 
Example:
#include ⋖ stdio.h ⋗

int main() {
    int i;

    for (i = 1; i <= 5; i++) {
        printf("Count: %d\n", i);
    }

    return 0;
}

2.while Loop: The while loop is used to repeatedly execute a block of code as long as a condition is true. The condition is evaluated before the execution of the loop. 
Example:
#include ⋖ stdio.h ⋗

int main() {
    int count = 1;

    while (count <= 5) {
       printf("Count: %d\n", count);
        count++;
    }

    return 0;
}

3.do-while Loop: The do-while loop is similar to the while loop, but the condition is evaluated after the execution of the loop, ensuring that the block of code is executed at least once. 
Example:
#include ⋖ stdio.h ⋗

int main() {
   int num;
do {
    printf("Enter a positive number:");
    scanf("%d", &num);
} while (num <= 0);
   printf("You entered a positive 
     number:%d\n", num);

    return 0;
}

These looping statements allow you to repeat a block of code based on certain conditions, enabling you to automate repetitive tasks and control the flow of your program.

preprocessor directives

Preprocessor directives in C are instructions that are processed before the actual compilation of the code. They start with a # symbol and are used to include header files, define macros, and perform conditional compilation. Here are some common preprocessor directives with simple examples for each.

#include Directive: The #include directive is used to include header files in your C program. Header files provide declarations for functions and macros, allowing you to use them in your code. 
Example: 
(using stdio.h for input/output functions):
#include ⋖ stdio.h ⋗

int main() {
    printf("Hello, World!\n");
    return 0;
}

#define Directive: The #define directive is used to create macros, which are symbolic names representing a value or code snippet. Macros can be used to make code more readable or to handle constants. 
Example (defining a constant value):
#include ⋖ stdio.h ⋗

#define PI 3.14159

int main() {
    double radius = 5.0;
    double area = PI * radius * radius;
    printf("Area of the circle: 
             %f\n",area);
    return 0;
}
These preprocessor directives are essential for managing the compilation process, making your code more modular, and improving code readability and maintainability.

Compilation Process in C Programming

The compilation process is the sequence of steps that transforms human-readable C source code into an executable program that a computer can run. It involves several stages to ensure code correctness and compatibility with the target system.

1.Preprocessing: Handles preprocessing directives (e.g., #include, #define). Expands macros and prepares the code for actual compilation.

2.Compilation: Translates C source code into assembly code. Checks for syntax errors and performs optimizations.

3.Assembly: Converts assembly code into machine code. Generates object files containing machine code for each source file.

4.Linking: Combines object files into a single executable file. Resolves references between object files and external libraries.

5.Executable: The final result of the compilation process. A program that can be run on the target system. The compilation process ensures that the C code you write can be executed efficiently and reliably, making it a crucial step in software development.
